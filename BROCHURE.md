## 📌 What is AuraGym OS?

**AuraGym OS** (branded as **AuraFarming**) is a modern, full-featured **SaaS gym management platform** designed to help fitness businesses of all sizes — from single-location gyms to multi-branch fitness chains — manage their entire operation from a single dashboard.

Built with a beautiful dark-themed UI, role-based access control, and real-time data powered by Supabase, AuraGym OS turns the chaos of gym administration into a streamlined, data-driven experience.

---

## 🎯 Who Is It For?

| Audience | Use Case |
|----------|----------|
| **Gym Owners & Chains** | Manage multiple gym locations, track revenue across branches, and oversee platform-wide operations from one dashboard |
| **Gym Administrators** | Handle day-to-day operations: member enrollment, trainer scheduling, payment collection, equipment maintenance |
| **Trainers** | View assigned members, track sessions, and maintain specialization profiles |
| **Members** | Self-service portal to view plans, check attendance history, see trainer details, and track membership status |

## ✨ Key Features

### 🏢 Multi-Gym Management (Super Admin)
- **Platform Overview Dashboard** — See total gyms, active gyms, and platform-level revenue at a glance
- **Gym Creation & Setup** — Create new gym locations with unique codes, branding colors, logos, and city assignments
- **Admin Assignment** — Create and assign gym-specific administrators during gym setup
- **Gym Status Tracking** — Monitor active/inactive status for every location

### 👥 Member Management
- **Complete Member Profiles** — Track name, phone, email, weight, status, and unique member codes
- **Plan & Trainer Assignment** — Assign membership plans and personal trainers to each member
- **Membership Status Tracking** — Active, expiring, and expired status indicators with due date tracking
- **Auto-Generated Codes** — System generates unique member codes (e.g., `KSMM0001`) based on gym codes

### 🏋️ Trainer Management
- **Trainer Profiles** — Track name, phone, specialization, salary, and unique trainer codes
- **Auto-Generated Codes** — Unique trainer codes (e.g., `KSTR0001`) tied to gym identifiers
- **Specialization Tracking** — Record each trainer's expertise areas
- **Salary Management** — Track trainer compensation

### ✅ Attendance Tracking
- **Dual Check-In Methods** — QR code scanning or manual member-code entry
- **Smart Check-In/Out** — System detects if a member is already checked in and automatically processes check-out
- **Real-Time Stats** — See today's check-ins and current in-gym count
- **Live Activity Feed** — View recent check-ins with timestamps, methods, and status

### 🔧 Equipment Management
- **Equipment Inventory** — Track all gym equipment with categories, brands, and models
- **Maintenance Scheduling** — Set next maintenance dates and track last service dates
- **Warranty Tracking** — Monitor warranty expiration for all equipment
- **Status Monitoring** — Operational, warning, critical, and maintenance-due status indicators
- **Cost Tracking** — Record purchase costs for financial planning

### 💰 Finance & Revenue
- **Revenue Dashboard** — Total revenue, net profit, pending dues, and expense totals
- **Payment Method Breakdown** — Visual pie chart showing UPI, cash, card, and online payment distribution
- **Payment History** — Complete payment records with dates, descriptions, methods, and status
- **Expense Management** — Record and categorize expenses (Payroll, Utilities, Maintenance, Rent, Operations)
- **Indian Currency Support** — Formatted in ₹ with Lakhs (L) and Thousands (K) notation

### 📊 Business Analytics
- **Member Churn Rate Trend** — Area chart showing monthly churn rate with trend analysis
- **Peak Hours Analysis** — Bar chart revealing gym usage patterns from 6 AM to 10 PM
- **Trainer Performance** — Member count, retention rates, satisfaction scores, and session data per trainer
- **Plan Profitability Rankings** — Revenue per plan, per-member value, and subscriber counts
- **Key Metrics** — Churn rate, retention rate, peak hours, and revenue growth at a glance

### 📋 Membership Plans
- **Plan Builder** — Create plans with custom names, prices, durations, and feature lists
- **Flexible Pricing** — Support for any duration (daily, monthly, quarterly, annual)
- **Feature Lists** — Define included features for each plan tier

### 🧑‍💻 Member Self-Service Portal
- **Personal Dashboard** — Membership status, days remaining, monthly visits, and plan info
- **Plan Details** — View current plan, pricing, features, and due date
- **Trainer Info** — See assigned trainer's name, specialization, and contact
- **Attendance History** — Full check-in/out history with dates, times, and methods

## 🔐 Role-Based Access Control

AuraGym OS uses a **4-tier role system** ensuring every user sees only what they need:

| Role | Access Level |
|------|-------------|
| **Super Admin** | Platform-wide: create gyms, assign admins, view all locations |
| **Gym Admin** | Single-gym: manage members, trainers, equipment, finance, attendance, analytics |
| **Trainer** | View assigned members and personal profile |
| **Member** | Self-service portal: view own plan, trainer, attendance |

- Route protection ensures unauthorized users cannot access restricted pages
- Role is determined at login via the `user_roles` database table
- Sidebar navigation adapts dynamically based on the active user role

### Database Schema (9 Core Tables)

| Table | Purpose | Key Fields |
|-------|---------|------------|
| `gyms` | Gym locations | name, code, city, email, colors, logo, is_active |
| `members` | Gym members | full_name, member_code, phone, email, status, weight, due_date |
| `trainers` | Gym trainers | full_name, trainer_code, phone, specialization, salary |
| `attendance` | Check-in/out records | member_id, gym_id, check_in, check_out, method |
| `equipment` | Gym equipment | name, category, brand, status, cost, warranty, maintenance |
| `payments` | Revenue records | member_id, amount, status, method, payment_date |
| `expenses` | Expense records | gym_id, category, amount, expense_date |
| `plans` | Membership plans | name, price, duration_days, features |
| `user_roles` | Role assignments | user_id, role, gym_id |

## 🚀 Quick Start

```bash
# Clone the repository
git clone <repo-url>
cd aura-gym-os

# Install dependencies
npm install

# Start development server
npm run dev
# → Open http://localhost:8080

# Build for production
npm run build

# Run tests
npm run test
```

### Prerequisites
- Node.js 18+
- npm or Bun package manager
- Supabase project (for backend)
